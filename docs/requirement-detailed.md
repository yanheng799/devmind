# DEVMIND 详细需求规格

## 项目概述

**项目名称**: DEVMIND - 基于新闻的A股股价预测Agent
**应用场景**: 自用研究辅助工具
**目标市场**: A股市场
**部署方式**: 本地运行

---

## 1. 功能需求

### 1.1 数据采集层

#### 1.1.1 新闻数据源
| 数据源 | 类型 | 采集方式 | 优先级 |
|-------|------|---------|--------|
| 东方财富 | 一级信源 | API/爬虫 | P0 |
| 同花顺公告 | 一级信源 | API/爬虫 | P0 |
| 新华财经 | 一级信源 | RSS/API | P0 |
| 央行/证监会公告 | 一级信源 | 官方RSS | P0 |
| 财经媒体解读 | 二级信源 | RSS/API | P1 |

**数据字段要求**:
- 标题、正文、发布时间
- 信源分类（一级/二级）
- 相关股票代码（如有）

#### 1.1.2 行情数据源
| 数据源 | 数据类型 | 频率 | 优先级 |
|-------|---------|-----|--------|
| AKShare | K线数据 | 分钟级 | P0 |
| AKShare | 财报数据 | 季度 | P0 |
| AKShare | 行业板块数据 | 日级 | P0 |

**数据字段要求**:
- OHLCV数据
- 成交量、成交额
- 涨跌幅

### 1.2 情报处理层

#### 1.2.1 事件提取（Event Extraction）
**输入**: 新闻正文
**输出格式**:
```json
{
  "event_type": "央行降息|公司财报|地缘政治|行业政策|其他",
  "entities": ["股票代码", "行业板块", "机构名称"],
  "magnitude": "25bp|同比增长10%|高|中|低",
  "timeframe": "即时|1-3日|1-4周|1-3月",
  "transmission_chain": ["降息→银行息差收窄→银行股承压"]
}
```

#### 1.2.2 情感分析
**输出格式**:
```json
{
  "sentiment_score": -0.7,
  "confidence": 0.85,
  "source_weight": 1.0
}
```

#### 1.2.3 历史相似事件检索
- 向量化存储历史事件（近10年）
- RAG召回Top-K相似案例
- 返回相似事件的涨跌结果统计

### 1.3 推理决策层

#### 1.3.1 ReAct Agent工具集
| 工具名称 | 功能 | 优先级 |
|---------|-----|--------|
| query_stock_price | 查询实时股价/历史K线 | P0 |
| query_financial_report | 查询财报数据 | P0 |
| query_industry_relation | 查询行业关联度 | P0 |
| query_market_sentiment | 查询市场情绪指数 | P1 |

#### 1.3.2 CoT推理结构
```
1. 事件分析
2. 多方论据（支撑上涨的因素）
3. 空方论据（支撑下跌的因素）
4. 历史案例参考
5. 综合判断
```

#### 1.3.3 置信度评估
**输出格式**:
```json
{
  "direction": "上涨|下跌|震荡",
  "probability": 0.68,
  "target_range": "+2%~+5%",
  "time_horizon": "1-3日",
  "risk_factors": ["政策落地不及预期", "市场情绪低迷"]
}
```

### 1.4 输出层

#### 1.4.1 预测报告
- 涨跌方向
- 预测幅度区间
- 时间窗口
- 置信度

#### 1.4.2 推理链透明化
- 关键依据
- 推理逻辑
- 核心假设

#### 1.4.3 风险提示
- 关键不确定因素
- 建议仓位/止损位（可选）

### 1.5 反馈优化层

#### 1.5.1 预测记录
- 每次预测存入数据库
- 记录时间戳、新闻ID、预测结果

#### 1.5.2 回测分析
- 按事件类型统计准确率
- 按行业统计准确率
- 按时间窗口统计准确率

## 2. 非功能需求

### 2.1 性能要求
| 指标 | 目标值 |
|-----|-------|
| 端到端推理延迟 | ≤3分钟 |
| 新闻采集频率 | 每5分钟 |
| 并发处理能力 | 10条新闻同时处理 |
| 数据库查询响应 | <100ms |

### 2.2 可用性要求
| 指标 | 目标值 |
|-----|-------|
| 系统可用性 | ≥95%（自用可接受） |
| 数据完整性 | 100% |
| 错误日志记录 | 完整记录 |

### 2.3 可维护性要求
- 模块化设计
- 配置文件管理
- 日志完整记录

## 6. 风险与应对

| 风险 | 应对措施 |
|-----|---------|
| 新闻源反爬 | 使用官方API，设置合理请求间隔 |
| API调用成本 | 设置调用限额，使用本地模型 |
| 预测准确性低 | 持续回测优化，添加人工校验 |
| 时效性问题 | 优先处理重大新闻，并行处理 |
